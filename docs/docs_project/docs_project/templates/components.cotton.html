Components are reusable pieces of view template. They can contain native django template syntax and can be used inside standard django templates. The only requirement is that both, the component and the view that contains the component must use a .cotton.html extension.

A minimal example:

{% cotton_verbatim %}{% verbatim %} {{ slot }} {% endverbatim %}{% endcotton_verbatim %} {% cotton_verbatim %}{% verbatim %}

Some content

 {% endverbatim %}{% endcotton_verbatim %}

Some content

The {% verbatim %}{{ slot }}{% endverbatim %} variable will contain all of the content provided between the opening and closing tag of the current component as defined in the parent.

Attributes

Components are highly configurable. One way to control the content and behaviour of a component is through attributes.

{% cotton_verbatim %}{% verbatim %}

It's {{ temperature }}{{ unit }} and the condition is {{ condition }}.

{% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %} It's 23c and the condition is windy.

Passing variables as attributes

Sometimes you'll want to pass a variable from the parent's context. To do so, prepend the attribute with a ":".

{% cotton_verbatim %}{% verbatim %} def weather_view(request): return render(request, 'view.cotton.html', { 'unit': get_weather_unit() }) {% endverbatim %}{% endcotton_verbatim %} {% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %}

Named slots

There are occasions when you will need to pass different types of content other than a simple string or variable to a component. In these cases, you can use named slots.

Named slots are defined with the {{ '...'|force_escape }} tag. The content is passed to the component like a standard template variable.

{% cotton_verbatim %}{% verbatim %}

{{ day }}:

{{ icon }} {{ label }}
{% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %}

Sunny

 {% endcotton_verbatim %}{% endverbatim %}

Tuesday:

Sunny

Using all attributes with {% verbatim %}{{ attrs }}{% endverbatim %}

It's sometimes useful to be able to print all provided attributes to the components in one "key=value" string. This is particularly useful when you are building form input components.

{% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %}

Boolean attributes

Sometimes you just want to pass a simple boolean to a component. Cotton supports providing the attribute name without a value which will provide a boolean True to the component.

{% cotton_verbatim %}{% verbatim %} {% if required is True %} * {% endif %} {% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %} *

Props

Props are defined using a c-props tag at the top of a component file. It can contain numerous key="value" pairs. It can also be a single key without a default value. Specifying an attribute as a 'prop' will remove the item from {% verbatim %}{{ attrs }}{% endverbatim %}.

Props give attributes default values

You may design a component that will often have a default behaviour and rarely needs overriding. In this case, you may opt to give a default value to your component.

{% cotton_verbatim %}{% verbatim %}
{{ slot }}
{% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %} All good! Oh no! {% endcotton_verbatim %}{% endverbatim %}
All good!
Oh no!

Props are excluded from {% verbatim %}{{ attrs }}{% endverbatim %}

In order to provide more control when using {{ attrs }} to pass down all attributes as a key=value string to a component. Keys defined as props will not be included in this string. This is beneficial when you want to pass any attribute to an html element inside a component using {% verbatim %}{{ attrs }}{% endverbatim %}, but also want to provide additional props for some other reason.

{% cotton_verbatim %}{% verbatim %} {% if errors %} {% for error in errors %} {{ error }} {% endfor %} {% endif %} {% endcotton_verbatim %}{% endverbatim %} {% cotton_verbatim %}{% verbatim %} {% endcotton_verbatim %}{% endverbatim %}
